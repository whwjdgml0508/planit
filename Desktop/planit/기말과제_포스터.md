# PlanIt: 생도 학습 관리 및 커뮤니티 통합 플랫폼

**조정민**  
*공군사관학교 컴퓨터과학과(졸업반)*

---

## 초 록

이 프로젝트는 **생도들의 학습 관리와 학과 커뮤니티를 하나로 통합**한 웹 기반 플랫폼 **PlanIt**을 개발하는 것을 목표로 한다. 기존의 학습 관리 시스템은 시간표, 플래너, 커뮤니티가 분산되어 있어 비효율적이었다. 본 연구에서는 Django 프레임워크를 기반으로 **올인원 통합 플랫폼**을 구축하여 학습 효율성을 향상시키고, PWA(Progressive Web App) 기술을 적용하여 **모든 기기에서 앱처럼 사용**할 수 있도록 하였다. 또한 **학과별 커뮤니티 기능**을 통해 정보 공유 및 협업을 강화하였다. 시스템은 AWS EC2에 배포되어 실제 운영 중이며, 사용자 피드백을 통해 지속적으로 개선되고 있다.

---

## 소 개

### 연구배경 및 필요성

생도들은 학습 관리를 위해 **여러 플랫폼을 동시에 사용**해야 하는 불편함을 겪고 있다. 시간표는 종이나 별도 앱으로 관리하고, 학습 계획은 노트에 기록하며, 학과 정보는 단체 채팅방에서 찾아야 한다. 이러한 **분산된 시스템**은 정보 손실, 시간 낭비, 협업 어려움을 초래한다. 또한 기존 학습 관리 앱들은 **생도의 특수한 환경**(학과별 커뮤니티, 군사 일정 등)을 고려하지 않아 실용성이 떨어진다.

### 연구목표

본 연구는 다음 네 가지 목표를 달성하고자 한다:

1. **통합 플랫폼 구축**: 시간표, 플래너, 커뮤니티를 하나의 플랫폼에 통합
2. **접근성 향상**: PWA 기술로 별도 설치 없이 모든 기기에서 사용 가능
3. **협업 강화**: 학과별 커뮤니티를 통한 정보 공유 및 소통 활성화
4. **실용성 검증**: 실제 배포 및 운영을 통한 사용자 피드백 수집

## 핵심 추진 전략

### 생체신호 측정부 (시간표 관리)

**목표**: 직관적이고 유연한 시간표 관리 시스템 구현

**구현 방법**:
- Django ORM을 활용한 `Subject` 및 `TimeSlot` 모델 설계
- 드래그 앤 드롭 기능을 위한 JavaScript 이벤트 핸들링
- Bootstrap Grid System을 활용한 반응형 시간표 레이아웃
- 색상 선택기(Color Picker)로 과목별 시각적 구분

**핵심 기술**:
- **Backend**: Django Models, Class-based Views
- **Frontend**: JavaScript Drag & Drop API, Fetch API
- **UI**: Bootstrap 5, Font Awesome Icons

### 지표분석 및 저장부 (학습 관리)

**목표**: 학습 진도를 체계적으로 추적하고 시각화

**구현 방법**:
- `Task`, `Goal`, `Progress` 모델로 학습 데이터 구조화
- Chart.js를 활용한 진도율 그래프 시각화
- 날짜별 필터링 및 과목별 통계 집계
- 목표 달성률 자동 계산 알고리즘

**핵심 기술**:
- **데이터 모델링**: Django ORM, Foreign Key 관계
- **시각화**: Chart.js, D3.js (예정)
- **통계 처리**: Django Aggregation, QuerySet API

### 전송부 (API 및 동기화)

**목표**: 안전하고 효율적인 데이터 전송 및 동기화

**구현 방법**:
- Django REST Framework로 RESTful API 구축
- JWT 토큰 기반 인증 시스템
- CORS 설정으로 크로스 도메인 요청 처리
- 파일 업로드 시 MIME 타입 검증 및 크기 제한

**핵심 기술**:
- **API**: Django REST Framework, SimpleJWT
- **보안**: CSRF Token, XSS Prevention, File Validation
- **동기화**: Session Management, Cookie Handling

### 복식부 (커뮤니티)

**목표**: 학과별 소통 공간 제공 및 정보 공유 활성화

**구현 방법**:
- `Post`, `Comment`, `Category` 모델로 게시판 구조 설계
- 권한 시스템으로 학과별 접근 제어
- 계층형 댓글 구조로 토론 지원
- 다중 파일 업로드 및 미리보기 기능

**핵심 기술**:
- **권한 관리**: Django Permissions, Custom Decorators
- **파일 처리**: Pillow, File Upload Validation
- **검색**: Django Q Objects, Full-text Search

### 시뮬레이션부 (PWA)

**목표**: 네이티브 앱과 같은 사용자 경험 제공

**구현 방법**:
- `manifest.json` 파일로 PWA 메타데이터 정의
- Service Worker로 오프라인 캐싱 구현
- 반응형 CSS Media Query로 다양한 화면 크기 대응
- 터치 이벤트 최적화로 모바일 UX 개선

**핵심 기술**:
- **PWA**: Service Worker, Cache API, Manifest
- **반응형**: Bootstrap Grid, CSS Flexbox, Media Queries
- **성능**: Lazy Loading, Code Splitting

## 프레임워크 서브 모듈

### 생체신호 측정부
- **시간표 관리**: 드래그 앤 드롭 방식의 직관적인 UI로 과목 추가/수정/삭제 가능
- **과목 커스터마이징**: 색상, 아이콘, 교수님 정보, 강의실 위치 등 상세 설정
- **자동 계산**: 학점 및 주간 수업 시간 자동 집계
- **다중 뷰**: 주간 시간표, 학기 전체 보기 등 다양한 시각화

### 지표분석 및 저장부
- **학습 플래너**: 날짜별 학습 목표 및 할 일 관리, 우선순위 설정
- **진도율 추적**: 과목별 학습 진행 상황을 실시간으로 시각화
- **통계 대시보드**: 주간/월간 학습 패턴 분석 및 성취도 그래프
- **목표 관리**: 단기/장기 학습 목표 설정 및 달성률 추적

### 전송부
- **REST API**: Django REST Framework 기반의 RESTful API 제공
- **실시간 동기화**: 모든 기기에서 데이터 자동 동기화
- **파일 업로드**: 최대 5개 파일 동시 업로드, 드래그 앤 드롭 지원
- **보안 통신**: CSRF 보호, XSS 방지, 안전한 세션 관리

### 복식부
- **커뮤니티 게시판**: 카테고리별(공지사항, 학습자료, 시험정보, 체력평가, 자유게시판) 게시판 운영
- **권한 관리**: 학과별 접근 권한 및 공지사항 작성 권한 차등 적용
- **댓글/답글 시스템**: 계층형 댓글로 활발한 토론 지원
- **검색/필터링**: 제목, 내용, 작성자 통합 검색 및 다양한 정렬 옵션

### 시뮬레이션부
- **PWA 지원**: 홈 화면에 추가하여 네이티브 앱처럼 사용 가능
- **오프라인 모드**: Service Worker를 통한 오프라인 기본 기능 지원
- **반응형 디자인**: PC, 태블릿, 모바일 모든 기기에 최적화
- **빠른 로딩**: 캐싱 전략으로 초기 로딩 시간 단축

## 결 론

본 연구를 통해 **생도 학습 관리 및 커뮤니티 통합 플랫폼 PlanIt**을 성공적으로 개발하고 배포하였다. 시간표, 플래너, 커뮤니티를 하나의 플랫폼에 통합함으로써 **학습 효율성을 크게 향상**시켰으며, PWA 기술 적용으로 **모든 기기에서 앱처럼 사용**할 수 있게 되었다. 특히 **학과별 커뮤니티 기능**은 정보 공유 및 협업을 활성화하여 생도들의 학습 환경을 개선하는 데 기여하였다.

현재 시스템은 **AWS EC2에서 안정적으로 운영** 중이며, 실제 사용자들로부터 긍정적인 피드백을 받고 있다. 향후 **HTTPS 적용, 실시간 알림, AI 학습 추천** 등의 기능을 추가하여 더욱 발전된 플랫폼으로 성장시킬 계획이다.
